### 方案3: 手动执行SQL语句
如果你想手动控制每一步，可以按以下顺序执行：
#### 步骤1: 删除旧索引
`sql
-- 删除 ZZ_PSUB_REF_diff_temp 旧索引

-- 删除 zz_pos_item_diff_temp 旧索引
DROP INDEX IDX_POS_DIFF_TEMP_DATE;
DROP INDEX IDX_POS_DIFF_TEMP_STAT;
`
#### 步骤2: 创建 ZZ_PSUB_REF_diff_temp 新索引
`sql
CREATE INDEX idx_diff_temp_trunc_date 
ON ZZ_PSUB_REF_diff_temp(TRUNC(create_date));
CREATE INDEX idx_diff_temp_export_main 
ON ZZ_PSUB_REF_diff_temp(TRUNC(create_date), compare_status, Subscriber, Cellular);
CREATE INDEX idx_diff_temp_status 
ON ZZ_PSUB_REF_diff_temp(compare_status);
CREATE INDEX idx_diff_temp_old_data 
ON ZZ_PSUB_REF_diff_temp(compare_status, Subscriber, Cellular) COMPRESS 1;
CREATE INDEX idx_diff_temp_new_data 
ON ZZ_PSUB_REF_diff_temp(compare_status, Subscriber_NB, Cellular_NB) COMPRESS 1;
`
#### 步骤3: 创建 zz_pos_item_diff_temp 新索引
`sql
CREATE INDEX idx_pos_diff_trunc_date 
ON zz_pos_item_diff_temp(TRUNC(create_date));
CREATE INDEX idx_pos_diff_export_main 
ON zz_pos_item_diff_temp(TRUNC(create_date), compare_status, subscriber, cellular, invoice_no);

ON zz_pos_item_diff_temp(compare_status);
CREATE INDEX idx_pos_diff_old_data 
ON zz_pos_item_diff_temp(compare_status, subscriber, cellular, invoice_no) COMPRESS 1;

ON zz_pos_item_diff_temp(compare_status, subscriber_nb, cellular_nb, invoice_no_nb) COMPRESS 1;

`sql

    ownname => USER, 
    tabname => 'ZZ_POS_ITEM_DIFF_TEMP', 
    cascade => TRUE

`

## ⚠️ 重要注意事项
### 1. 执行时机
- 建议在**低峰期**执行（如夜间或周末）
- 在删除旧索引到创建新索引之间，查询性能会临时下降
### 2. 数据安全
- ✅ **表中数据不会丢失**
- ✅ 只是删除和重建索引
- ✅ 建议先在**测试环境**验证
### 3. 回滚方案
如果迁移后出现问题，可以回滚到旧索引：
`sql
-- 删除新索引
DROP INDEX idx_diff_temp_trunc_date;
DROP INDEX idx_diff_temp_export_main;
DROP INDEX idx_diff_temp_status;
DROP INDEX idx_diff_temp_old_data;
DROP INDEX idx_diff_temp_new_data;
DROP INDEX idx_pos_diff_trunc_date;
DROP INDEX idx_pos_diff_export_main;
DROP INDEX idx_pos_diff_status;
DROP INDEX idx_pos_diff_old_data;
DROP INDEX idx_pos_diff_new_data;
-- 重建旧索引
CREATE INDEX IDX_ZZ_PSUB_REF_DIFF_TEMP_STAT ON ZZ_PSUB_REF_diff_temp(COMPARE_STATUS);
CREATE INDEX IDX_POS_DIFF_TEMP_DATE ON zz_pos_item_diff_temp(CREATE_DATE);
CREATE INDEX IDX_POS_DIFF_TEMP_STAT ON zz_pos_item_diff_temp(COMPARE_STATUS);
`

## 📞 问题排查
### 问题1: 删除索引时报错 "ORA-01418: 指定的索引不存在"
**原因:** 旧索引可能已经被删除或索引名不同
**解决:** 忽略此错误，继续执行后续步骤
---
### 问题2: 创建索引时报错 "ORA-00955: 名称已由现有对象使用"
**原因:** 新索引已经存在
**解决:** 
`sql
-- 删除已存在的索引
DROP INDEX idx_diff_temp_trunc_date;
-- 然后重新创建
CREATE INDEX idx_diff_temp_trunc_date ON ZZ_PSUB_REF_diff_temp(TRUNC(create_date));
`
---
### 问题3: 查询未使用新索引
**原因:** 统计信息未更新
**解决:**
`sql
-- 重新收集统计信息
EXEC DBMS_STATS.GATHER_TABLE_STATS(ownname => USER, tabname => 'ZZ_PSUB_REF_DIFF_TEMP', cascade => TRUE);
`
