## 🎯 失效原因总结

### 为什么 `idx_pos_diff_new_data` 对你的查询失效？

```
根本原因: 索引列顺序与查询条件不匹配

索引设计:
(compare_status, subscriber_nb, cellular_nb, invoice_no_nb)
      ①              ②              ③              ④

你的查询条件:
- TRUNC(create_date) ❌ 不在索引中
- compare_status ✅ 在第①列
- subscriber_nb ❌ WHERE中没有（中断！）
- cellular_nb ❌ WHERE中没有
- invoice_no_nb ✅ 在第④列（但无法使用，因为跳过了②③）

结论:
1. 缺少 TRUNC(create_date)（最重要的过滤条件）
2. 缺少 subscriber_nb 和 cellular_nb（中断了索引链）
3. ORDER BY 使用 subscriber 而不是 subscriber_nb（不匹配）

这就是为什么只能用第1列的原因！
```
