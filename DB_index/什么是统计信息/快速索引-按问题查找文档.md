## 🎯 核心知识点速记卡

### 卡片1：CRUD触发机制
```
INSERT/UPDATE/DELETE → 累计>10% → 标记过期 → 等维护窗口 → 自动收集
TRUNCATE/SQL*Loader → 不触发 → 永远不自动收集 → 必须手动收集
```

### 卡片2：统计信息过期后果
```
执行计划不失效 ❌ 但会生成错误计划
查询仍成功 ✅ 但性能下降10-100倍
数据正确 ✅ 但系统可能雪崩
```

### 卡片3：必做操作
```
TRUNCATE + 批量导入后 → 立即手动收集统计信息 ✅
否则 → 性能灾难 ❌
```

### 卡片4：手动收集命令
```sql
-- Oracle
EXEC DBMS_STATS.GATHER_TABLE_STATS('FES', 'TABLE_NAME', cascade=>TRUE);

-- MySQL
ANALYZE TABLE table_name;
```
