# 索引最左原则详解与实战分析
## 📚 第一部分：索引基础概念
### 1. 什么是复合索引（组合索引）？
复合索引是包含多个列的索引，例如：
```sql
CREATE INDEX idx_example ON table_name(col1, col2, col3);
```
这个索引在物理存储上是按照 **col1 → col2 → col3** 的顺序排列的。
**类比理解：** 就像字典的目录
- 第一层：按拼音首字母 (A, B, C...)
- 第二层：按第二个字母 (AA, AB, AC...)
- 第三层：按第三个字母 (AAA, AAB, AAC...)
---
### 2. 什么是最左前缀原则（最左匹配原则）？
**核心规则：** 复合索引 (col1, col2, col3) 可以支持以下查询：
| 查询条件 | 是否使用索引 | 原因 |
|----------|-------------|------|
| WHERE col1 = ? | ✅ 完全匹配 | 使用第一列 |
| WHERE col1 = ? AND col2 = ? | ✅ 完全匹配 | 使用前两列 |
| WHERE col1 = ? AND col2 = ? AND col3 = ? | ✅ 完全匹配 | 使用所有列 |
| WHERE col2 = ? | ❌ **不使用** | 缺少第一列 col1 |
| WHERE col3 = ? | ❌ **不使用** | 缺少第一列 col1 |
| WHERE col1 = ? AND col3 = ? | ⚠️ 部分使用 | 只使用 col1，跳过 col2 |
| WHERE col2 = ? AND col3 = ? | ❌ **不使用** | 缺少第一列 col1 |
---
### 🎯 关键规则总结
> **必须从索引的第一列开始，连续匹配，不能跳过中间列**
**例子：**
```sql
-- 索引: (A, B, C, D)
✅ WHERE A = 1                          -- 使用 A
✅ WHERE A = 1 AND B = 2                -- 使用 A, B
✅ WHERE A = 1 AND B = 2 AND C = 3      -- 使用 A, B, C
✅ WHERE A = 1 AND B = 2 AND C = 3 AND D = 4  -- 使用 A, B, C, D
⚠️ WHERE A = 1 AND C = 3                -- 只使用 A（跳过了B）
⚠️ WHERE A = 1 AND D = 4                -- 只使用 A（跳过了B, C）
❌ WHERE B = 2                          -- 不使用索引（缺少A）
❌ WHERE C = 3                          -- 不使用索引（缺少A）
❌ WHERE B = 2 AND C = 3                -- 不使用索引（缺少A）
```

## 💡 第五部分：TRUNC函数索引详解
### Oracle中的TRUNC函数索引
#### ✅ Oracle完全支持
```sql
-- ✅ 创建函数索引
CREATE INDEX idx_trunc_date ON table_name(TRUNC(create_date));
-- ✅ 使用函数索引（必须完全匹配）
SELECT * FROM table_name WHERE TRUNC(create_date) = TRUNC(SYSDATE);
-- 会使用索引
-- ❌ 不使用函数索引
SELECT * FROM table_name WHERE create_date = SYSDATE;
-- 不会使用索引（表达式不匹配）
SELECT * FROM table_name WHERE TO_CHAR(create_date, 'YYYYMMDD') = TO_CHAR(SYSDATE, 'YYYYMMDD');
-- 不会使用索引（不同的函数）
```
**关键点：**
- ✅ 查询中的函数表达式必须**完全一致**
- ✅ WHERE TRUNC(create_date) = ? 会使用索引
- ❌ WHERE create_date >= ? 不会使用 TRUNC 函数索引
---
### MySQL中的TRUNC函数索引
#### ⚠️ MySQL 8.0+ 支持，但语法不同
```sql
-- MySQL 8.0+ 函数索引语法
CREATE INDEX idx_date_day ON table_name((DATE(create_date)));
-- 或者使用虚拟列（推荐）
ALTER TABLE table_name ADD COLUMN create_date_day DATE AS (DATE(create_date)) VIRTUAL;
CREATE INDEX idx_date_day ON table_name(create_date_day);
-- 查询时使用
SELECT * FROM table_name WHERE DATE(create_date) = CURDATE();
```
#### ❌ MySQL 5.7及以下不支持函数索引
**替代方案：**
```sql
-- 方案1: 增加虚拟列（推荐）
ALTER TABLE table_name ADD COLUMN create_date_day DATE;
-- 在插入/更新时维护该列
INSERT INTO table_name (create_date, create_date_day, ...) 
VALUES (NOW(), DATE(NOW()), ...);
-- 创建普通索引
CREATE INDEX idx_date_day ON table_name(create_date_day);
-- 查询时使用
SELECT * FROM table_name WHERE create_date_day = CURDATE();
-- 方案2: 使用范围查询
SELECT * FROM table_name 
WHERE create_date >= CURDATE() AND create_date < CURDATE() + INTERVAL 1 DAY;
-- 这样可以使用 create_date 上的普通索引
```

## 🆚 第九部分：Oracle vs MySQL 索引对比
| 特性 | Oracle | MySQL (InnoDB) | 说明 |
|------|--------|----------------|------|
| **复合索引** | ✅ 支持 | ✅ 支持 | 语法相同 |
| **最左前缀原则** | ✅ 适用 | ✅ 适用 | **原理完全相同** |
| **函数索引** | ✅ 支持 | ⚠️ 8.0+支持 | MySQL 5.7不支持 |
| **索引压缩** | ✅ COMPRESS | ⚠️ 不同机制 | MySQL使用前缀压缩 |
| **覆盖索引** | ✅ 隐式支持 | ✅ 支持 | MySQL需指定 |
### Oracle函数索引
```sql
-- Oracle语法
CREATE INDEX idx_trunc ON table_name(TRUNC(date_col));
-- 查询
SELECT * FROM table_name WHERE TRUNC(date_col) = TRUNC(SYSDATE);
```
### MySQL函数索引
```sql
-- MySQL 8.0+语法
CREATE INDEX idx_date ON table_name((DATE(date_col)));
-- 查询
SELECT * FROM table_name WHERE DATE(date_col) = CURDATE();
-- MySQL 5.7替代方案（虚拟列）
ALTER TABLE table_name ADD COLUMN date_only DATE AS (DATE(date_col)) STORED;
CREATE INDEX idx_date ON table_name(date_only);
SELECT * FROM table_name WHERE date_only = CURDATE();
```

## 🎯 第十部分：索引失效的常见场景
### ❌ 场景1: 在索引列上使用函数（没有函数索引）
```sql
-- 索引: CREATE INDEX idx ON table(create_date);
-- ❌ 索引失效
SELECT * FROM table WHERE TRUNC(create_date) = TRUNC(SYSDATE);
SELECT * FROM table WHERE DATE_FORMAT(create_date, '%Y-%m-%d') = '2026-04-02';
SELECT * FROM table WHERE YEAR(create_date) = 2026;
-- ✅ 使用索引
SELECT * FROM table WHERE create_date >= TRUNC(SYSDATE) AND create_date < TRUNC(SYSDATE) + 1;
```
**解决方案:** 创建函数索引
```sql
CREATE INDEX idx_trunc ON table(TRUNC(create_date));
```
---
### ❌ 场景2: 违反最左前缀原则
```sql
-- 索引: CREATE INDEX idx ON table(col1, col2, col3);
-- ❌ 索引失效（缺少第一列）
SELECT * FROM table WHERE col2 = 100;
SELECT * FROM table WHERE col3 = 200;
SELECT * FROM table WHERE col2 = 100 AND col3 = 200;
-- ✅ 使用索引
SELECT * FROM table WHERE col1 = 1;
SELECT * FROM table WHERE col1 = 1 AND col2 = 100;
SELECT * FROM table WHERE col1 = 1 AND col2 = 100 AND col3 = 200;
```
---
### ❌ 场景3: 使用 OR 连接不同列
```sql
-- 索引: CREATE INDEX idx ON table(col1, col2);
-- ❌ 索引失效
SELECT * FROM table WHERE col1 = 1 OR col2 = 2;
-- ✅ 使用索引（改用 UNION）
SELECT * FROM table WHERE col1 = 1
UNION ALL
SELECT * FROM table WHERE col2 = 2 AND col1 != 1;
```
---
### ❌ 场景4: 使用 != 或 <> 操作符
```sql
-- 索引: CREATE INDEX idx ON table(status);
-- ❌ 可能不使用索引
SELECT * FROM table WHERE status != 'DELETED';
SELECT * FROM table WHERE status <> 'DELETED';
-- ✅ 使用索引
SELECT * FROM table WHERE status IN ('ACTIVE', 'PENDING', 'CLOSED');
```
---
### ❌ 场景5: 使用 LIKE '%keyword%'
```sql
-- 索引: CREATE INDEX idx ON table(name);
-- ❌ 索引失效（前导通配符）
SELECT * FROM table WHERE name LIKE '%John%';
SELECT * FROM table WHERE name LIKE '%John';
-- ✅ 使用索引
SELECT * FROM table WHERE name LIKE 'John%';
```
---
### ❌ 场景6: 隐式类型转换
```sql
-- 索引: CREATE INDEX idx ON table(phone_number);  -- VARCHAR2类型
-- ❌ 索引失效（隐式转换）
SELECT * FROM table WHERE phone_number = 12345678;  -- 数字类型
-- ✅ 使用索引
SELECT * FROM table WHERE phone_number = '12345678';  -- 字符串类型
```

## 📊 第十二部分：索引设计决策表
### 如何决定创建哪些索引？
| 查询模式 | 推荐索引 | 示例 |
|----------|----------|------|
| **单列等值查询** | 单列索引 | CREATE INDEX idx ON t(col1) |
| **多列等值查询** | 复合索引（按选择性排序） | CREATE INDEX idx ON t(col1, col2) |
| **范围查询** | 范围列放最后 | CREATE INDEX idx ON t(col1, col2_range) |
| **函数查询** | 函数索引 | CREATE INDEX idx ON t(TRUNC(date)) |
| **覆盖查询** | 包含SELECT列 | CREATE INDEX idx ON t(where_cols, select_cols) |
| **高重复率列** | 使用COMPRESS | CREATE INDEX idx ON t(status, id) COMPRESS 1 |
### 索引列顺序决策
**优先级排序（从高到低）：**
1. **等值查询列** (=) - 放在最前面
2. **排序列** (ORDER BY) - 放在中间
3. **范围查询列** (>, <, BETWEEN) - 放在最后
4. **选择性高的列** - 放在前面（过滤效果好）
**例子：**
```sql
-- 查询: WHERE status = 'ACTIVE' AND user_id = 100 AND amount > 1000 ORDER BY create_date
-- ✅ 好的索引设计
CREATE INDEX idx ON orders(
    user_id,        -- 等值查询，选择性高（放第1）
    status,         -- 等值查询（放第2）
    create_date,    -- ORDER BY（放第3）
    amount          -- 范围查询（放最后）
);
-- ❌ 不好的索引设计
CREATE INDEX idx ON orders(amount, status, user_id);  -- 范围查询放前面
```
