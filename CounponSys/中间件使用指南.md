# 中间件使用指南

> ⚠️ **重要**: 5-10W QPS场景必须使用Kafka，ActiveMQ扛不住！

## 〇、中间件选型总结

| 中间件 | 场景 | 吞吐量 | 本项目使用 |
|--------|------|--------|-----------|
| **Redis** | 缓存/库存/分布式锁 | 10W+ QPS | ✅ 必须 |
| **Kafka** | 高并发消息队列 | 100W+ QPS | ✅ 推荐 |
| ActiveMQ | 低并发消息队列 | 1-2W QPS | ❌ 备用 |

## 一、Redis 使用说明

### 1. 安装后需要做什么？
**什么都不用做！** 代码会自动操作Redis。

### 2. 数据是怎么进去的？

| 场景 | 数据Key | 谁写入 | 什么时候写 |
|------|---------|--------|-----------|
| 秒杀库存 | `seckill:stock:{activityId}` | 定时任务 | 活动开始前10分钟自动预热 |
| 用户已抢次数 | `seckill:user:{activityId}:{userId}` | Lua脚本 | 用户抢购成功时自动记录 |
| 每日领取计数 | `coupon:claim:daily:{userId}` | Service代码 | 用户领券成功时自动累加 |
| 券模板缓存 | `coupon:template:{id}` | Service代码 | 查询时自动缓存 |

### 3. 数据流程图

```
秒杀场景:
┌─────────────────────────────────────────────────────────────┐
│  定时任务(活动开始前10分钟)                                    │
│  └─→ 从数据库读取活动库存 → 写入Redis                          │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  用户抢购请求                                                 │
│  └─→ 执行Lua脚本 → 原子扣减Redis库存 + 记录用户已抢次数         │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  MQ消费者(异步)                                               │
│  └─→ 扣减数据库库存 + 创建优惠券记录                           │
└─────────────────────────────────────────────────────────────┘
```

## 二、Kafka 使用说明 (推荐)

### 1. 为什么用Kafka而不是ActiveMQ?
| 指标 | ActiveMQ | Kafka | 你的场景 |
|------|----------|-------|---------|
| 吞吐量 | 1-2万/秒 | 10-100万/秒 | 5-10万QPS |
| 结论 | ❌ 扛不住 | ✅ 绰绰有余 | **必须Kafka** |

### 2. Topic清单

| Topic名称 | 分区数 | 用途 | 生产者 | 消费者 |
|-----------|-------|------|--------|--------|
| `coupon.seckill.order` | 10 | 秒杀订单 | SeckillServiceImpl | SeckillOrderKafkaConsumer |
| `coupon.grant` | 5 | 优惠券发放 | MonthlyGrantJob | CouponGrantConsumer |
| `coupon.consumption.event` | 5 | 消费事件 | 计费系统(外部) | ConsumptionEventListener |

### 3. 消息流程

```
秒杀订单处理:
┌─────────────────────────────────────────────────────────────┐
│  SeckillServiceImpl.grabCoupon()                            │
│  └─→ Redis扣库存成功 → 创建订单 → Kafka发送消息               │
│                                                              │
│      kafkaTemplate.send("coupon.seckill.order", orderId)    │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  SeckillOrderKafkaConsumer (10个并发消费者)                  │
│  └─→ 扣积分 → 发放优惠券 → 更新订单状态                       │
│  └─→ 失败时: 回滚Redis库存 + 回滚积分                        │
└─────────────────────────────────────────────────────────────┘
```

### 4. 快速启动Kafka

```bash
# Docker方式启动
docker run -d --name zookeeper -p 2181:2181 zookeeper:3.7
docker run -d --name kafka -p 9092:9092 \
  -e KAFKA_ZOOKEEPER_CONNECT=host.docker.internal:2181 \
  -e KAFKA_ADVERTISED_LISTENERS=PLAINTEXT://localhost:9092 \
  wurstmeister/kafka

# 验证
docker exec -it kafka kafka-topics.sh --list --bootstrap-server localhost:9092
```

## 二(备用)、ActiveMQ 使用说明

### 1. 安装后需要做什么？
**什么都不用做！** Spring Boot会自动创建队列。

### 2. 队列是怎么创建的？
代码里定义的队列名称，ActiveMQ会在第一次使用时自动创建：

| 队列名称 | 用途 | 生产者 | 消费者 |
|---------|------|--------|--------|
| `coupon.grant.queue` | 优惠券发放 | MonthlyGrantJob | CouponGrantConsumer |
| `seckill.order.queue` | 秒杀订单处理 | SeckillServiceImpl | SeckillOrderConsumer |
| `coupon.expire.queue` | 优惠券过期 | 预留 | 预留 |
| `consumption.event.queue` | 消费事件 | 计费系统(外部) | ConsumptionEventListener |

### 3. 消息流程图

```
月度套餐赠送流程:
┌─────────────────────────────────────────────────────────────┐
│  MonthlyGrantJob (每月1号凌晨2点触发)                         │
│  └─→ 查询所有套餐赠送模板 → 发送MQ消息                         │
│                                                              │
│      jmsTemplate.convertAndSend("coupon.grant.queue", msg)  │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  CouponGrantConsumer (@JmsListener自动监听)                  │
│  └─→ 查询符合条件用户 → 逐个发放优惠券                         │
└─────────────────────────────────────────────────────────────┘
```

```
秒杀订单处理流程:
┌─────────────────────────────────────────────────────────────┐
│  SeckillServiceImpl.grabCoupon()                            │
│  └─→ Redis扣库存成功 → 创建秒杀订单 → 发送MQ消息               │
│                                                              │
│      jmsTemplate.convertAndSend("seckill.order.queue", id)  │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  SeckillOrderConsumer (@JmsListener自动监听)                 │
│  └─→ 扣积分 → 发放优惠券 → 更新订单状态                        │
│  └─→ 失败时回滚积分                                          │
└─────────────────────────────────────────────────────────────┘
```

## 三、对接计费系统

### 消费激励型优惠券需要计费系统发消息

计费系统需要在用户消费后发送MQ消息：

```java
// 计费系统代码示例
jmsTemplate.convertAndSend("consumption.event.queue", JSON.toJSONString(new HashMap<String, Object>() {{
    put("userId", "U123456");
    put("userMobile", "85212345678");
    put("amount", 100.00);           // 本次消费金额
    put("monthlyTotal", 520.00);     // 本月累计消费
    put("consumeType", "VOICE");     // 消费类型
    put("consumeTime", "2026-03-25 10:30:00");
}}));
```

优惠券系统的 `ConsumptionEventListener` 会自动监听处理。

## 四、快速验证步骤

### 1. 启动中间件
```bash
# Redis (Docker方式)
docker run -d --name redis -p 6379:6379 redis

# ActiveMQ (Docker方式)
docker run -d --name activemq -p 61616:61616 -p 8161:8161 apache/activemq-classic
# 管理界面: http://localhost:8161  账号: admin/admin
```

### 2. 修改配置
编辑 `application.yml`，确保连接信息正确：
```yaml
spring:
  redis:
    host: localhost
    port: 6379
  activemq:
    broker-url: tcp://localhost:61616
```

### 3. 启动应用
运行 `CouponSystemApplication.main()`

### 4. 访问API文档
打开浏览器: http://localhost:8090/doc.html

### 5. 验证Redis
```bash
# 连接Redis CLI
redis-cli

# 查看所有key
keys *

# 查看秒杀库存
get seckill:stock:1
```

### 6. 验证ActiveMQ
打开 http://localhost:8161/admin/queues.jsp 查看队列状态
