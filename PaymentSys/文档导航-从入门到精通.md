## 🔑 关键概念速记

### 核心公式
```
标准版 = 数据库幂等性
       ↓
单机场景够用（TPS<500）

增强版 = Redis幂等性 + 数据库幂等性
       ↓
分布式/高并发必须（TPS>1000）
```

### 问题根源
```
问题1: 时间戳每次不同 → 幂等键不同 → 重复创建
问题2: 并发窗口长 → 同时查询 → 都创建
问题3: 数据库锁竞争 → CPU 100% → 系统崩溃
```

### 解决方案
```
方案1: 前端传UUID → 幂等键固定 ✅
方案2: Redis分布式锁 → 并发窗口=0 ✅
方案3: Redis回调去重 → 快速拦截 ✅
```
