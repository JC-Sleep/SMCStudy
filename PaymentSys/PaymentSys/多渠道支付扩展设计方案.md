# 支付系统多渠道扩展设计方案

## 文档概述

本文档是《支付系统设计方案-渣打银行对接》的扩展版本，详细说明如何基于策略模式设计可扩展的多渠道支付网关架构，支持对接支付宝、建设银行等多个支付渠道。

## 架构设计

### 2. 核心组件

#### 2.1 PaymentGateway 接口

所有支付渠道必须实现此接口，定义统一的支付能力：

```java
public interface PaymentGateway {
    // 获取渠道标识
    PaymentChannel getChannel();
    String getChannelName();
    
    // 核心支付能力
    GatewayPaymentResponse createPayment(PaymentInitRequest request, String transactionId);
    GatewayTransactionStatus queryTransactionStatus(String gatewayTransactionId);
    boolean closeOrder(String gatewayTransactionId);
    GatewayRefundResponse refund(RefundRequest request);
    
    // 回调处理
    boolean verifyCallback(String rawBody, String signature, Map<String, String> headers);
    PaymentCallbackData parseCallbackData(String rawBody);
    
    // 渠道能力
    boolean isAvailable();
    int getPriority();
    boolean supportsPaymentMethod(String paymentMethod);
}
```

#### 2.2 AbstractPaymentGateway 抽象基类

提供通用实现，子类只需实现特定逻辑：

```java
public abstract class AbstractPaymentGateway implements PaymentGateway {
    // 配置获取
    protected abstract String getApiEndpoint();
    protected abstract String getApiKey();
    protected abstract String getApiSecret();
    
    // 请求构建
    protected abstract Map<String, Object> buildPaymentRequest(...);
    protected abstract String buildSignature(Map<String, Object> data);
    
    // 响应解析
    protected abstract GatewayPaymentResponse parsePaymentResponse(String body);
    protected abstract GatewayTransactionStatus parseStatusResponse(String body);
    
    // 通用HTTP方法
    protected String doPost(String url, Map<String, Object> data, Map<String, String> headers);
    protected String doGet(String url, Map<String, String> headers);
}
```

#### 2.3 PaymentGatewayRouter 路由器

```java
@Component
public class PaymentGatewayRouter {
    // Spring自动注入所有实现
    @Autowired
    private List<PaymentGateway> gateways;
    
    // 根据渠道获取网关
    public PaymentGateway getGateway(PaymentChannel channel);
    
    // 根据支付方式智能选择
    public PaymentGateway selectGateway(String paymentMethod);
    
    // 获取所有可用渠道
    public List<PaymentGateway> getAvailableGateways();
}
```

## 各渠道对接指南

### 1. 渣打银行 (Standard Chartered)

#### 1.1 接入准备
1. 联系渣打银行获取商户资质
2. 申请API密钥（API Key 和 API Secret）
3. 配置回调白名单IP
4. 提供回调URL

#### 1.2 配置项
```yaml
scb:
  api:
    endpoint: https://api.standardchartered.com/payment
    key: ${SCB_API_KEY}
    secret: ${SCB_API_SECRET}
```

#### 1.3 签名方式
- **算法**: HMAC-SHA256
- **签名串**: 参数按字母排序后拼接

#### 1.4 回调格式
```json
{
  "gatewayTransactionId": "SCB123456789",
  "orderReference": "CSP001234567",
  "amount": 176.00,
  "status": "SUCCESS",
  "paymentTime": "2026-02-03T20:02:30Z"
}
```

### 2. 支付宝 (Alipay)

#### 2.1 接入准备
1. 注册支付宝开放平台账号
2. 创建应用获取AppID
3. 配置RSA2密钥对（商户私钥 + 支付宝公钥）
4. 签约所需产品（网页支付、APP支付等）

#### 2.2 配置项
```yaml
alipay:
  gateway:
    url: https://openapi.alipay.com/gateway.do
  app:
    id: ${ALIPAY_APP_ID}
  merchant:
    private:
      key: ${ALIPAY_MERCHANT_PRIVATE_KEY}
  public:
    key: ${ALIPAY_PUBLIC_KEY}
  sign:
    type: RSA2
```

#### 2.3 签名方式
- **算法**: RSA2 (SHA256withRSA)
- **私钥**: 商户私钥签名
- **公钥**: 支付宝公钥验签

#### 2.4 API接口
| 接口 | 方法名 | 说明 |
|-----|-------|------|
| 统一收单交易创建 | alipay.trade.page.pay | 电脑网站支付 |
| 统一收单交易查询 | alipay.trade.query | 查询订单状态 |
| 统一收单交易关闭 | alipay.trade.close | 关闭订单 |
| 统一收单交易退款 | alipay.trade.refund | 申请退款 |

#### 2.5 回调格式（form表单）
```
trade_no=2016112121001004000000000001
out_trade_no=CSP001234567
total_amount=176.00
trade_status=TRADE_SUCCESS
gmt_payment=2026-02-03 20:02:30
sign=xxx
sign_type=RSA2
```

#### 2.6 状态映射
| 支付宝状态 | 系统状态 |
|-----------|---------|
| WAIT_BUYER_PAY | PENDING |
| TRADE_SUCCESS | SUCCESS |
| TRADE_FINISHED | SUCCESS |
| TRADE_CLOSED | FAILED |

### 3. 建设银行 (CCB)

#### 3.1 接入准备
1. 与建设银行签署电子商务协议
2. 获取商户代码（MERCHANTID）
3. 获取柜台代码（POSID）
4. 获取分行代码（BRANCHID）
5. 获取加密密钥

#### 3.2 配置项
```yaml
ccb:
  api:
    endpoint: https://ibsbjstar.ccb.com.cn/CCBIS/B2CMainPlat_00_LINKAGE
  merchant:
    id: ${CCB_MERCHANT_ID}
    key: ${CCB_MERCHANT_KEY}
  pos:
    id: ${CCB_POS_ID}
  branch:
    id: ${CCB_BRANCH_ID}
```

#### 3.3 签名方式
- **算法**: MD5
- **签名串**: 固定顺序参数拼接 + KEY

#### 3.4 请求参数
```
MERCHANTID=商户代码
POSID=柜台代码
BRANCHID=分行代码
ORDERID=订单号
PAYMENT=金额
CURCODE=01（人民币）
TXCODE=530550（网关支付）
MAC=签名
```

#### 3.5 回调格式
```
POSID=xxx&BRANCHID=xxx&ORDERID=xxx&PAYMENT=xxx&SUCCESS=Y&MAC=xxx
```

## 统一API接口

### 1. 发起支付

**接口**: `POST /api/payment/initiate`

**请求**:
```json
{
  "orderReference": "CSP001234567",
  "userId": "33xxxx",
  "amount": 176.00,
  "currency": "HKD",
  "paymentMethod": "ALIPAY",
  "channel": "ALIPAY",        // 可选，不指定则自动选择
  "returnUrl": "https://shop.example.com/return"
}
```

**响应**:
```json
{
  "code": 0,
  "message": "成功",
  "data": {
    "transactionId": "TXN20260203200150001",
    "paymentUrl": "https://openapi.alipay.com/gateway.do?...",
    "status": "PENDING",
    "channel": "ALIPAY",
    "channelName": "支付宝",
    "expiryTime": "2026-02-03T20:11:50Z"
  }
}
```

### 2. 查询支付状态

**接口**: `GET /api/payment/status/{transactionId}`

### 3. 退款

**接口**: `POST /api/payment/refund`

**请求**:
```json
{
  "transactionId": "TXN20260203200150001",
  "refundAmount": 50.00,
  "refundReason": "客户申请退款"
}
```

## 常见问题排查

### Q1: 新渠道回调收不到？
1. 检查回调URL是否正确配置到渠道后台
2. 检查服务器防火墙是否放行渠道IP
3. 查看回调日志表 `PAYMENT_CALLBACK_LOG`

### Q2: 签名验证失败？
1. 检查密钥配置是否正确
2. 检查签名算法是否匹配
3. 打印签名原文对比

### Q3: 渠道切换不生效？
1. 确认渠道枚举已添加
2. 确认网关类已注入Spring容器
3. 检查 `isAvailable()` 方法是否返回true

## 总结

本方案通过策略模式实现了可扩展的多渠道支付架构：

1. **统一接口**: `PaymentGateway` 定义标准支付能力
2. **抽象基类**: `AbstractPaymentGateway` 提供通用实现
3. **智能路由**: `PaymentGatewayRouter` 自动选择最优渠道
4. **配置驱动**: 新渠道只需实现接口+配置即可接入
5. **向后兼容**: 保持原有渣打银行接口不变

扩展新渠道只需：
1. 实现 `PaymentGateway` 接口
2. 添加配置项
3. 无需修改现有代码
